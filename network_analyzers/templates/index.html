Ciclo DORA (DHCP)

El protocolo DHCP sigue este ciclo obligatorio para asignar IPs:

1️⃣ DISCOVER – Cliente emite solicitud de IP (broadcast)
2️⃣ OFFER – Servidor responde con una IP disponible
3️⃣ REQUEST – Cliente confirma y solicita esa IP
4️⃣ ACK – Servidor confirma la asignación (ciclo exitoso)
📊 Métricas Analizadas para Detectar Problemas:
Tiempos por fase:
  • t_discover_ack_s – Tiempo total DISCOVER → ACK
  • t_offer_request_s – Tiempo OFFER → REQUEST
  • t_request_ack_s – Tiempo REQUEST → ACK
Condiciones de error:
  • ❌ DISCOVER sin OFFER → servidor no responde
  • ❌ REQUEST sin ACK → servidor no confirma
  • ❌ Presencia de NAK → conflicto de IP / rechazo
  • ⏱️ Ciclo DORA lento → t_discover_ack_s > umbral (10s)
